# Research

Use this skill when a PM asks for desk research, competitor context, market context, standards, or external technical background.

Workflow:
1. Browse the web for current, relevant sources.
2. Prefer primary sources: official docs, standards bodies, vendor docs, filings, engineering blogs, or reputable analyst/news sources when primary sources are unavailable.
3. Summarize findings with citations and date accessed.
4. Save structured findings with `save_research_packet(topic, findings, citations)`.
5. Read `report-format://research_brief` if the user wants a written artifact.

Keep external research separate from internal customer evidence.
